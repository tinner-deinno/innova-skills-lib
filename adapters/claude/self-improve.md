# /self-improve — ทักษะพัฒนาตัวเองอัตโนมัติ

> "เราเรียนรู้จากการทำ เราจำจากการสำเร็จ เราพัฒนาจากการแก้ไข"

## คืออะไร

ทักษะนี้ไม่ใช่แค่การทำงาน แต่เป็นวงจรการเรียนรู้ที่พัฒนาตัวเองได้:
- **เรียนรู้** จากเป้าหมายและบริบท
- **ทดสอบ** ด้วยการดำเนินการขั้นตอน
- **จำ** สิ่งที่ทำได้ถูกต้อง
- **แก้ไข** สิ่งที่ทำผิดพลาด
- **เสริม** เป็นวิธีการที่ดีขึ้นสำหรับครั้งถัดไป

## การใช้งาน

```
/self-improve --goal "เป้าหมายที่ต้องการบรรลุ"
/self-improve --goal "สร้างระบบ multi-agent" --context "ใช้ ollama.mdes" --max-attempts 3
/self-improve --history  # ดูประวัติการพัฒนาตัวเอง
/self-improve --reset    # รีเซ็ตประวัติการเรียนรู้ (ใช้ด้วยความระมัดระวัง)
```

## วิธีการทำงาน (PDCA Cycle พร้อมการเรียนรู้ตนเอง)

### 1. Plan (วางแผน)
- ตรวจสอบประวัติการทำงานที่คล้ายกัน
- หากมี: ใช้วิธีการที่ดีที่สุดจากประวัติ
- หากไม่มี: ใช้ `/learn` เพื่อสำรวจโดเมน แล้วแบ่งเป้าหมายเป็นขั้นตอนย่อย

### 2. Do (ดำเนินการ)
- ดำเนินการแต่ละขั้นตอนโดยใช้ทักษะที่เหมาะสม (เช่น `/thread`, `/learn`, `/taskcreate`)
- บันทึกผลลัพธ์แต่ละขั้นตอน (สำเร็จ/ล้มเหลว พร้อมเหตุผล)

### 3. Check (ตรวจสอบ)
- ประเมินผลรวม: สำเร็จตามเป้าหมายหรือไม่?
- ระบุจุดที่ทำได้ดีและจุดที่ต้องปรับปรุง
- จัดเรียงบทเรียนตามความสำคัญ

### 4. Act (ปรับปรุงและทำซ้ำ)
- บันทึกประสบการณ์นี้ลงในประวัติการเรียนรู้
- หากเป้าหมายเกี่ยวกับการพัฒนาตนเองของทักษะนี้ ให้ปรับปรุงวิธีการทำงานของทักษะนี้
- เตรียมพร้อมสำหรับครั้งถัดไปด้วยความรู้ที่เพิ่มขึ้น

## ตัวอย่างการพัฒนาตนเอง

ครั้งที่ 1: ใช้วิธีการ A → สำเร็จ 60% → บทเรียน: ขั้นตอนที่ 3 ล้มเหลวเพราะขาดข้อมูล
ครั้งที่ 2: ใช้วิธีการ A + เพิ่มขั้นตอนตรวจสอบข้อมูล → สำเร็จ 80% → บทเรียน: วิธีการตรวจสอบข้อมูลได้ผล
ครั้งที่ 3: ใช้วิธีการ A + วิธีการตรวจสอบข้อมูลที่ปรับปรุงแล้ว → สำเร็จ 95% → บทเรียน: วิธีการตรวจสอบข้อมูลเวอร์ชัน 2 ได้ผลดีกว่า

ในครั้งที่ 4 ระบบจะเลือกใช้วิธีการที่ดีที่สุดจากประวัติโดยอัตโนมัติ

## หลักการทำงาน

1. **ไม่รีบเร่ง** - วงจรการเรียนรู้ต้องใช้เวลา
2. **ไม่ตัดสิน** - สะท้อนผลลัพธ์โดยไม่โทษตัวเองหรือทักษะ
3. **เรียนรู้จากทุกผลลัพธ์** - ทั้งสำเร็จและล้มเหลวล้วนมีบทเรียน
4. **พัฒนาตนเองได้จริง** - ทักษะนี้สามารถปรับปรุงวิธีการทำงานของตัวเองได้
5. **ภาษาตามผู้ใช้** - หากผู้ใช้ใช้ภาษาไทย ทักษะนี้จะสื่อสารด้วยภาษาไทย

## ตัวอย่างการใช้งานจริง

### เป้าหมาย: สร้างระบบ multi-agent แม่ลูกที่ทำงานได้จริง
```bash
/self-improve --goal "สร้างระบบ multi-agent แม่ลูกที่สามารถสั่งการและเห็นผลได้" --context "ใช้ ollama.mdes multiagentแก๊งค์ที่มีอยู่" --max-attempts 5
```

สิ่งที่จะเกิดขึ้น:
1. ทักษะนี้จะตรวจสอบประวัติ หากไม่มี จะใช้ `/learn` เพื่อสำรวจ multi-agent-gangs
2. แบ่งเป้าหมายเป็นขั้นตอน: สำรวจโครงสร้าง → สร้างแม่ agent → สร้างลูก agent → ทดสอบการสั่งการ → ตรวจสอบผล
3. ดำเนินการแต่ละขั้นตอน
4. บันทึกผลและบทเรียน
5. หากยังไม่ถึงเป้าหมาย จะปรับปรุงวิธีการและลองใหม่ (สูงสุด 5 ครั้ง)
6. ในครั้งถัดไปที่มีเป้าหมายคล้ายกัน จะใช้วิธีการที่ดีที่สุดจากประวัติโดยอัตโนมัติ

## สิ่งที่ทักษะนี้ให้คุณได้

- ระบบที่เรียนรู้จากความผิดพลาดโดยไม่ต้องสั่งให้แก้ไขทุกครั้ง
- วิธีการทำงานที่ดีขึ้นเรื่อยๆ ตามประสบการณ์จริง
- บทเรียนที่ถูกจัดเก็บและสามารถทวนดูได้ทุกเมื่อ
- ความมั่นใจว่าความพยายามแต่ละครั้งจะทำให้คุณและระบบของคุณเก่งขึ้น
- เครื่องมือที่ช่วยให้คุณบรรลุเป้าหมายที่ยากได้จริงๆ
